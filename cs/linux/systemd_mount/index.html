Mount a filesystem

tldr description of mount

tldr mount

mount
Provides access to an entire filesystem in one directory.
- Show all mounted filesystems:
    mount
- Mount a device to a directory:
    mount -t filesystem_type path/to/device_file path/to/target_directory
- Mount a CD-ROM device (with the filetype ISO9660) to /cdrom (readonly):
    mount -t iso9660 -o ro /dev/cdrom /cdrom
- Mount all the filesystem defined in /etc/fstab:
    mount -a
- Mount a specific filesystem described in /etc/fstab (e.g. "/dev/sda1 /my_drive ext2 defaults 0 2"):
    mount /my_drive

Use CLI to mount a filesystem

For instance, mount a AWS EFS filesystem to your EC2 instance /files folder

mount -t nfs -o nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2 fs-XXXXXX.efs.us-west-2.amazonaws.com:/ /files

Use systemd to mount a filesystem

Requirement: the filename of the mount file should be the same as the Where field.

# files.mount
[Unit]
Description=Mount The File System
After=network.target

[Mount]
What=fs-XXXXX.efs.us-west-2.amazonaws.com:/
Where=/files
Type=nfs
Options=auto

[Install]
WantedBy=multi-user.target

Start the service

# In ubuntu, move it to /etc/systemd/system/
cp files.mount /etc/systemd/system/
systemctl daemon-reload
systemctl start files.mount

Mount as a specific user

If I want to mount the filesystem as the ubuntu user, specify it in the Options field.

[Unit]
Description=Mount The File System
After=network.target

[Mount]
What=fs-XXXXXX.efs.us-east-2.amazonaws.com:/
Where=/files
Type=nfs
Options=nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2,uid=$(id -u ubuntu),gid=$(id -g ubuntu)

[Install]
WantedBy=multi-user.target

Mount as a normal user

Found this option can be used for sudo user:

nosuid

Disables the set-user-identifier and set-group-identifier bits. This prevents remote users from gaining higher privileges by running a setuid program.

[Unit]
Description=Mount The File System
After=network.target

[Mount]
What=fs-XXXXXX.efs.us-east-2.amazonaws.com:/
Where=/files
Type=nfs
Options=nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2,nosuid

[Install]
WantedBy=multi-user.target

Using /etc/fstab

We can mount the remote NFS shares automatically at boot by adding them to /etc/fstab file on the client.

# /etc/fstab entry
. . .
203.0.113.0:/var/nfs/general    /nfs/general   nfs auto,nofail,noatime,nolock,intr,tcp,actimeo=1800 0 0
203.0.113.0:/home       /nfs/home      nfs auto,nofail,noatime,nolock,intr,tcp,actimeo=1800 0 0
