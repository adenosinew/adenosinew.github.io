Overview:

There are tons of tutorial of git. For those basic concepts, I am going to link to some good tutorial. This one is focusing on some real life use case. What if you encounter these errors or what if I want to do this…

Update a branch (feature) with different branch (develop)

git checkout develop
git pull
git checkout feature

git merge develop
# or  
git rebase develop

The difference between with git rebase is that rebase keep all commits history from your branch, and that is important if your partial commits have a lot of content that can be interesting to keep. This option is obligatory in some teams.

Remove grey submodule

git rm --cached <folder_name>

git add .
git commit -m "<your_message>"
git push --all

git merge
